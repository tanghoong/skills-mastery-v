# Chapter 5 Drill — Write and Test a CLAUDE.md

## Steps
1. Read this repo's own `CLAUDE.md` as a model (purpose, layout, commands, conventions — all concise).
2. For your own project, draft a `CLAUDE.md` containing at least:
   - One-line purpose.
   - Test / build / lint / dev commands.
   - Three conventions, with emphasis (`NEVER`, `ONLY`, etc.) on the important ones.
3. Start a **fresh session** and ask: "Based on my project instructions, what conventions should you follow here?"
4. If it can't recite your rules accurately, tighten them and repeat.

## Reflection
- Which of your rules did Claude follow cleanly, and which got ignored or softened? What made the difference in wording?
- What did you *almost* put in `CLAUDE.md` that actually belongs in a per-task prompt instead (Ch. 5 §4)?
- What's one personal preference that belongs in `~/.claude/CLAUDE.md` rather than the shared project file?
