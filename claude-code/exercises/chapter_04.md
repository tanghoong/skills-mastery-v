# Chapter 4 Drill — Run the Full Core Loop

Pick a real, small change in a real project. Run the loop deliberately, narrating each phase.

## Steps
1. **EXPLORE** — "Read the files involved in `<feature>` and explain how it works. Don't write code yet." Verify the explanation matches reality.
2. **PLAN** — "Propose a plan to `<change>`. List files + changes. Wait for approval." Then **change one thing** about the plan before approving.
3. **CODE** — Approve. Watch the first edit. Ask it to run tests/types after.
4. **COMMIT** — "Commit with a clear message; nothing unrelated."
5. `/clear`.

## Reflection
- At which step did you catch something you'd otherwise have missed?
- How big was your review burden at the PLAN step vs. what it would have been reviewing a finished diff?
- Did Claude try to start coding before you wanted? If so, what phrasing would have kept it in explore/plan?
