# Chapter 7 Drill — Plan Mode & the Control Slider

## Steps
1. **Plan mode:** Run a moderately complex request in plan mode. Read the proposed plan, change one thing, then approve. Notice it made *no* edits until you approved.
2. **Accept-edits:** Switch to accept-edits for a small, well-scoped task and feel the difference in flow.
3. **Allow-list:** Pre-approve one safe command you run constantly (e.g. your test command) and confirm the permission prompts for it stop.

(Use `/help` or the docs to find how your version toggles modes.)

## Reflection
- How much smaller was your review burden reviewing the *plan* vs. a finished diff?
- For each of your last three real tasks, where on the slider (plan → ask → accept-edits → bypass) should each have been, and why?
- What's one action you'd *never* put on an allow-list, and why?
