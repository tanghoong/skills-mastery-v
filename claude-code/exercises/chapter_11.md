# Chapter 11 Drill — Package a Procedure as a Skill

## Steps
1. Ask Claude what skills are currently available, or inspect your environment's skill list. Read one skill's "when to use" description to see how the trigger is phrased.
2. Identify a multi-step procedure you've explained to Claude **more than once**.
3. Draft a `SKILL.md`: a sharp "Use when…" description + concrete step-by-step instructions (apply Ch. 3 to the body).
4. Run a task that *should* trigger it and see whether Claude picks it up on its own.

## Reflection
- Did the skill trigger automatically? If not, how would you sharpen the "when to use" description?
- Why is this better as a skill than as extra lines in `CLAUDE.md`? (Tie to progressive disclosure, Ch. 11 §2.)
- What supporting file or template could you bundle to make the skill even more reliable?
