# TaskFlow — Project Specification

## What Problem It Solves

Teams lose track of work. TaskFlow is a **team task and project management API** — the backend for a Trello/Linear-style application. It is deliberately scoped to force every hard Express pattern: auth, RBAC, real-time, background jobs, file storage, caching, and observability.

This is not a toy CRUD app. Every architectural decision has a real reason behind it.

---

## Domain Model

```
Organization ─── has many ──► Projects
     │
     └── has many ──► OrganizationMembers (userId + role)

Project ─── has many ──► Tasks
              │
              └── has many ──► Labels

Task ─── has many ──► Comments
  │  ─── has many ──► Attachments
  │  ─── has many ──► TaskLabels
  └── belongs to ──► assignee (User)

ActivityLog ── records every mutation on any entity
Notification ── per-user, generated by domain events
```

### Entities

```typescript
// Branded IDs — prevent passing a ProjectId where a TaskId is expected
type OrgId         = number & { readonly __brand: "OrgId" };
type ProjectId     = number & { readonly __brand: "ProjectId" };
type TaskId        = number & { readonly __brand: "TaskId" };
type UserId        = number & { readonly __brand: "UserId" };
type CommentId     = number & { readonly __brand: "CommentId" };
type AttachmentId  = number & { readonly __brand: "AttachmentId" };
type LabelId       = number & { readonly __brand: "LabelId" };

type OrgRole       = "owner" | "admin" | "member" | "viewer";
type TaskStatus    = "backlog" | "todo" | "in_progress" | "in_review" | "done" | "cancelled";
type TaskPriority  = "urgent" | "high" | "medium" | "low" | "none";
```

## API Surface (v1)

### Auth
```
POST   /api/v1/auth/register
POST   /api/v1/auth/login
POST   /api/v1/auth/refresh
POST   /api/v1/auth/logout
GET    /api/v1/auth/me
```

### Organizations
```
POST   /api/v1/orgs
GET    /api/v1/orgs/:orgId
PATCH  /api/v1/orgs/:orgId
DELETE /api/v1/orgs/:orgId

GET    /api/v1/orgs/:orgId/members
POST   /api/v1/orgs/:orgId/members
PATCH  /api/v1/orgs/:orgId/members/:userId
DELETE /api/v1/orgs/:orgId/members/:userId
```

### Projects
```
GET    /api/v1/orgs/:orgId/projects
POST   /api/v1/orgs/:orgId/projects
GET    /api/v1/orgs/:orgId/projects/:projectId
PATCH  /api/v1/orgs/:orgId/projects/:projectId
DELETE /api/v1/orgs/:orgId/projects/:projectId
```

### Tasks
```
GET    /api/v1/projects/:projectId/tasks      ?status=&priority=&assignee=&page=&limit=
POST   /api/v1/projects/:projectId/tasks
GET    /api/v1/projects/:projectId/tasks/:taskId
PATCH  /api/v1/projects/:projectId/tasks/:taskId
DELETE /api/v1/projects/:projectId/tasks/:taskId

POST   /api/v1/projects/:projectId/tasks/:taskId/comments
GET    /api/v1/projects/:projectId/tasks/:taskId/comments
DELETE /api/v1/projects/:projectId/tasks/:taskId/comments/:commentId

POST   /api/v1/projects/:projectId/tasks/:taskId/attachments
DELETE /api/v1/projects/:projectId/tasks/:taskId/attachments/:attachmentId
```

### Activity & Notifications
```
GET    /api/v1/orgs/:orgId/activity
GET    /api/v1/me/notifications
PATCH  /api/v1/me/notifications/:notificationId/read
```

### WebSocket
```
WS     /ws   (auth via token query param or Authorization header)
Events emitted: task.created, task.updated, task.deleted,
                comment.created, member.joined, member.left
```

### Health & Docs
```
GET    /health
GET    /api-docs   (Swagger UI)
```

## Response Envelope

Every response uses a consistent typed envelope:

```typescript
// Success
{
  "ok": true,
  "data": { ... },
  "meta": { "page": 1, "limit": 20, "total": 143 }  // only on lists
}

// Error
{
  "ok": false,
  "error": {
    "code": "TASK_NOT_FOUND",
    "message": "Task 42 does not exist in project 7",
    "statusCode": 404
  }
}
```

## Production Architecture (Railway)

```
Railway Project: taskflow
├── Service: api          ← Node.js, built from Dockerfile
├── Service: postgres     ← Railway managed PostgreSQL 16
└── Service: redis        ← Railway managed Redis 7

Environment variables (set in Railway dashboard, never in code):
  DATABASE_URL            ← injected by Railway postgres plugin
  REDIS_URL               ← injected by Railway redis plugin
  JWT_ACCESS_SECRET       ← manually set
  JWT_REFRESH_SECRET      ← manually set
  AWS_S3_BUCKET           ← for file attachments
  AWS_ACCESS_KEY_ID       ← for S3
  AWS_SECRET_ACCESS_KEY   ← for S3
  NODE_ENV=production
  PORT=3000
```
